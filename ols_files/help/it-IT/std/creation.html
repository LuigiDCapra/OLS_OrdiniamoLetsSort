Creazione di nuove collezioni di dati.

Le collezione di dati possono essere create in vari modi:
  • scrivendo del codice in JavaScript;
  • usando l'interfaccia grafica del programma OLS;
  • impiegando un comune editor di testi per inserire i dati di nostro interesse in un file.

Creazione di collezioni di dati in JavaScript

Premessa

Come dice Niklaus Wirth la programmazione concerne la stesura di algoritmi e la gestione di strutture dati, le collezioni di dati esistono già naturalmente all'interno dei programmi. Spesso  non si tratta di "crearle" ex novo, ma di renderle accessibili in modo che si possa operare facilmente su di esse semplificando il lavoro di debug e/o l'analisi dei dati da parte dell'operatore.
Si potrebbe pensare che gli strumenti in dotazione ai vari browser siano già più che sufficiente, senonché basta uno sguardo ai rispettivi debugger per accorgersi che sono stati progettati per cercare i difetti presenti nel codice, ma non per consentire l'esplorazione delle strutture dati complesse presenti al loro interno.
I normali debugger risultano spesso inadeguati. quando si tratta di accedere a strutture dati complesse e di navigare all'interno di esse.

Il programma OLS si propone di offrire una soluzione migliore minimizzando la fatica dell'utente grazie all'impiego di una interfaccia grafica e di strumenti specifici.
Supponiamo di voler impiegare le risorse del programma DDJ per studiare il contenuto di una generica struttura dati già definita all'interno di un programma JavaScript, ad esempio:

var S_aFld_aRcd4 = [
  {"szNm":"sz0",    "szType":"string",  "iLen":8, "szRem":"string zero"},
  {"szNm":"iNum_X", "szType":"number",  "iLen":8, "szRem":"iNum_X"},
  {"szNm":"sz_Y",   "szType":"string",  "iLen":8, "szRem":"sz_Y"}
];

var S_aRcd4  =
[
["anatra_0", 0, "a"],
["cavallo",  2, "c"],
["gallo",    4, "e"],
["lupo",     6, "g"],
["rana",     8, "i"]
];
Per poter usare gli strumenti grafici della libreria DDJ nella pagina Web che si intende debuggare si dovrà innanzitutto "importare" il codice degli oggetti  richiesti inserendo gli opportuni elementi script, ad esempio
<script src="ddj.js"></script>
Nel caso in cui si intenda modificare il codice di OLS non occorre fare nulla perché gli oggetti richiesti per usare gli strumenti grafici sono già importati.

Le strutture dati che si intendono studiare dovranno quindi essere "registrate" nel framework creando i corrispondenti oggetti JavaScript, vedi ad esempio:
new CL_XDB0("S_aFld_aRcd4",  C_JKndTup_aObj, S_aFld_aRcd4, G_aFld_aFld,  "G_aFld_aFld",  "", C_WwFlag_Null); 
new CL_XDB("S_aRcd4",        C_JKndTup_aRcd, S_aRcd4,      S_aFld_aRcd4, "S_aFld_aRcd4", "", C_WwFlag_Null);
dopodiché si potrà operare sulle collezioni mediante gli strumenti grafici messi a disposizione da OLS.

Si noti a questo riguardo che per quanto concerne il programma OLS è indifferente che le collezioni che si desiderano esplorare siano state generate al suo interno da una procedura scritta in JavaScript o siano state caricate da un file esterno, ciò che conta è che siano state regolarmente registrate creando gli oggetti corrispondenti.

Supponiamo ad esempio di voler "registrare" la struttura dati costituita dall'oggetto JavaScript così definito:
var S_aRcd4  = [["anatra", 0, "a"], ["cavallo", 2, "c"], ["gallo", 4, "e"], ["lupo", 6, "g"], ["rana", 8, "i"] ];   (1)
La struttura dati S_aRcd4 è chiaramente un array di array contenente dati di tipo eterogeneo, sebbene memorizzati secondo una certa logica.
Sebbene la struttura S_aRcd4 contenga al suo interno un "miscuglio" di dati di tipo diverso: stringhe, numeri interi, caratteri alfabetici, si ravvisa un elemento di regolarità che può essere evidenziato riscrivendola come segue:
var S_aRcd4  = [		(2)
["anatra",   0, "a"],
["cavallo",  2, "c"],
["gallo",    4, "e"],
["lupo",     6, "g"],
["rana",     8, "i"]
];
Si osservi che i codici (1) e (2)  sono assolutamente identici per quanto concerne l'interprete JavaScript nel senso che ad essi corrisponderà lo stesso codice; non così per gli esseri umani poiché il secondo codice evidenzia degli elementi di regolarità che aiutano a comprendere l'organizzazione della struttura dati.
In riferimento al codice riorganizzato sotto forma di tabella (2) diremo record le righe della tabella e campi le colonne della stessa.
Si osservi che sono possibili delle nomenclature alternative per cui i record sono detti talvolta anche righe o tuple  e i campi colonne, attributi, proprietà. o caratteristiche.

Gli elementi di regolarità di cui si è parlato possono essere esplicitati scrivendone una descrizione formale che prende il nome di tracciato record.

Il tracciato record delle righe della collezione S_aRcd4 può essere rappresentato come segue:

nome campo tipo di dato lunghezza massima commento
szNm szType iLen szRem
sz0 string
 8
 esempio di stringa
iNum_X number
 8
 esempio di numero
sh0 string
 8
 esempio di carattere

E' interessante osservare che la tabella precedente è a sua volta un array di array pertanto può essere rappresentata secondo le convenzioni di JavaScript nel modo seguente:
var S_aFld_aRcd4 = [
  {"szNm":"sz0",    "szType":"string",  "iLen":8, "szRem":"esempio di stringa"},
  {"szNm":"iNum_X", "szType":"number",  "iLen":8, "szRem":"iNum_X"},
  {"szNm":"ch0",    "szType":"string",  "iLen":8, "szRem":"sz_Y"}
];
Fatto che consente di registrare il tracciato record di una collezione assieme alla collezione stessa:
new CL_XDB0("S_aFld_aRcd4",  C_JKndTup_aObj, S_aFld_aRcd4, G_aFld_aFld,  "G_aFld_aFld",  "", C_WwFlag_Null); 
new CL_XDB("S_aRcd4",        C_JKndTup_aRcd, S_aRcd4,      S_aFld_aRcd4, "S_aFld_aRcd4", "", C_WwFlag_Null);
Si osservi che nella creazione dei suddetti oggetti si impiegano due costruttori diversi che fanno riferimento a classi differenti.
Il tracciato record S_aFld_aRcd4, corrispondente ad tracciato record della collezione S_aRcd4, viene registrato ricorrendo al costruttore della classe CL_XDB0, mentre si impiega il costruttore della classe CL_XDB  per registrare la collezione dati S_aRcd4.
La differenza consiste nel fatto che il primo costruttore si limita a creare un oggetto XDB descrivente la collezione presente in memoria (Layout fisico), mentre il secondo crea anche una vista utente, inoltre iscrive la collezione considerata nella lista delle collezioni dati presenti in memoria.

Le funzioni corrispondenti ai due costruttori sono caratterizzate dalla stessa lista di parametri: il primo parametro corrisponde al nome della collezione. Il secondo definisce la tipologia della collezione. il terzo è la collezione vera e propria che si intende registrare, seguono la struttura dati corrispondente al tracciato record e il nome della stessa, infine si ha la possibilità di inserire un commento e di specificare dei flag che definiscono alcune proprietà speciali della collezione.
Si osservi che il tracciato record G_aFld_aFld (cui si fa riferimento nella la registrazione del tracciato record S_aFld_aRcd4 della collezione S_aRcd4), 
new CL_XDB0("S_aFld_aRcd4",  C_JKndTup_aObj, S_aFld_aRcd4, G_aFld_aFld,  "G_aFld_aFld",  "", C_WwFlag_Null);
è già definito poiché è un tracciato record standard, pertanto non è necessario registrarlo.

Ricorso alla opzione AutoDetect.

Nel caso in cui non si volesse registrare una collezione di cui non si dispone tracciato record la libreria DDJ consente di generarlo automaticamente ricorrendo alla opzione AutoDetect richiedendo al programma DDJ di costruirsela da solo, in maniera automatica, analizzando i dati della prima riga della collezione.
Per attivare la suddetta funzionalità è basta inserire la stringa "AutoDetect" al posto del nome del Layout come evidenziato in rosso nell'esempio.

new CL_XDB("S_aUsrView_N", C_JKndTup_aRcd, CL_aUsrView_N.S_aUsrView_N, null, "AutoDetect",  "", C_WwFlag_Null);

Creazione di collezioni di dati utilizzando l'interfaccia grafica di OLS.


Il programma OLS prevede diverse procedure per la creazione di Collezioni di dati:
  • creazione di una Collezione completamente nuova (New);
  • creazione di una collezione mediante calcolo dei valori di una funzione (Function);
  • stesura del codice JavaScript in una finestra del browser (TextArea);
  • inserimento di dati in un file;
  • riferimento ad una variabile globale presente in memoria.

Creazione di una Collezione di Dati completamente nuova.

Il procedimento per la creazione di una collezione di dati completamente nuova prevede:
  • la scelta del nome della nuova collezione di dati,
  • la selezione del formato e del tracciato record della collezione fra quelli già esistenti o la creazione di un nuovo tracciato record,
  • il caricamento dei dati.

Creazione di un nuovo Tracciato Record.

Nel caso in cui il tracciato record della nuova collezione non fosse ancora presente in memoria bisognerà crearlo.
Essendo il tracciato record di una collezione a sua volta una collezione non si tratterà che di creare e inizializzare la collezione dei dati che descrivono i campi della nuova collezione.

Clicchiamo sul tasto di selezione e caricamento Load Collection e poi su Source. Selezioniamo la voce New come mostrato in Figura 1.


Si aprirà una finestra che offre la possibilità di specificare il nome della nuova collezione (Collection Name) e di selezionare il nome del tracciato record della nuova collezione (Record Layout (aFld)).
Nel caso in cui il tracciato record desiderato non fosse già disponibile in memoria si può richiedere il preventivo caricamento del nuovo tracciato record selezionando lo speciale valore "New Layout". Vedi Figura 2.

Creazione di una nuova Collezione facendo riferimento ad Tracciato Record già esistente.

Volendo creare una nuova collezione di dati facendo riferimento ad un tracciato record già presente in memoria sarà sufficiente
  • cliccare sulla voce Select Collection della barra dei menu,
  • selezionare la voce New della lista Source,
  • inserire il nome della nuova collezione nel campo Collection Name,
  • selezionare il formato in cui si desidera siano rappresentati i dati scegliendolo nella lista Kind,
  • selezionare il tracciato record dalla lista Record layout (aFld),
  • e confermare.
  • Potrete quindi procedere al caricamento dei dati della nuova collezione.

Creazione di un nuovo Layout

Selezionando  "Record Layout aFld" = "New Layout" si forza la creazione della collezione corrispondente al nuovo tracciato record.
Per quest'ultima OLS selezionerà automaticamente il formato aObj e il tracciato record G_aFld_aFld che descrive l'organizzazione dei tracciati record.



Per terminare il caricamento basta cliccare sulla "X" che compare nell'angolo in alto a destra della finestra di dialogo.

Terminato il caricamento dei valori del tracciato record il programma OLS domanderà se si procederà alla creazione della collezione desiderata e all'inserimento dei valori degli elementi.


Previa conferma verrà avviata la procedura di caricamento aprendo una scheda vuota conforme al tracciato record della nuova collezione.



Completato il caricamento dei dati nella scheda l'operatore confermerà l'inserimento della scheda nella collezione cliccando sul bottone Confirm posto alla base della scheda.


Il programma OLS inserirà i dati nella Collezione poi aprirà un'altra scheda vuota.

Per completare il caricamento basta cliccare sulla X posta nell'angolo in alto a destra della scheda di input.
Terminato il caricamento delle schede il programma OLS mostrerà i valori della Collezione di dati caricata sotto forma di Tabella allo scopo di consentire all'operatore di ricontrollare i valori inseriti.






Caricamento dei dati nella nuova collezione.

Confermata la creazione di una nuova collezione apparirà una tabella vuota che presenta semplicemente le intestazioni delle colonne corrispondenti ai diversi campi dei record (attributi delle tuple), per introdurre un nuovo record bisognerà pertanto utilizzare la procedura normalmente prevista per l'aggiunta di nuovi record alle collezioni.
Cliccando sulla voce New Tuple della barra dei menu si aprirà una scheda che consentirà di introdurre i valori del nuovo record.
Introducete i dati e confermate cliccando sul bottone Confirm.
Apparirà quindi una nuova scheda vuota che vi offrirà la possibilità di introdurre i valori di un nuovo record e così via.
Per interrompere il caricamento basterà cliccare semplicemente sulla X nell'angolo in alto a destra.

Creazione di una nuova collezione di dati mediante calcolo dei valori di una funzione

Clicchiamo sul tasto di selezione e caricamento Load Collection e poi su Source.
Selezioniamo la voce Function come mostrato in Figura 1.


Specifichiamo il valore minimo e massimo dell'intervallo sul quale si desiderano calcolare i valori della funzione e il passo di incremento.
Inseriamo il codice JavaScript della funzione che desideriamo calcolare (è consentito omettere il nome della libreria ovvero Math.).
Indicate eventualmente il nome della nuova collezione, il formato e il Layout e confermate.
