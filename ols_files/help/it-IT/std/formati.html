Schemi e Formati.

Rappresentazioni dei dati in memoria.

Il linguaggio di programmazione JavaScript consente di rappresentare le strutture dati con una varietà di schemi diversi, molti dei quali possono essere salvati in formato JSON senza alcuna modifica.
La nomenclatura che ci accingiamo ad introdurre ha lo scopo di distinguere fra i diversi schemi/ formati supportati da OLS.

Null.


Formato nullo o illegale.

Arr.


Array di valori di tipo omogeneo.

Diremo array delle collezioni ordinate, di dati semplici, omogenei cui è possibile accedere utilizzando l'operatore parentesi quadre [], specificando l'indice corrispondente alla posizione degli elementi all'interno della struttura dati.


Esempio:


var S_ai   = [41,70,53,19,37,21,6,115,84,72];

var S_asz  = ["rosso", "verde", "blu"];

as_

Array associativi.
Diremo array associativi delle collezioni non ordinate, di dati semplici, omogenei cui è possibile accedere utilizzando la notazione puntata o l'operatore parentesi quadre [], specificando il valore numerico una chiave costituita da una stringa di caratteri.

Esempio:

var S_Settimana = {"lunedì": 1, "martedì":2, "mercoledì":3,  "giovedì":4, "venerdì":5, "sabato":6, "domenica":7};

Obj.


Oggetti.

Diremo oggetti definiti dall'utente (user defined object) delle collezioni non ordinate, di dati di tipo arbitrario cui è possibile accedere utilizzando la notazione puntata o l'operatore parentesi quadre [], specificando una chiave costituita da una stringa di caratteri.


Esempio:


var S_Obj = {sz0:"S_Obj", iNum1:111, sz2:"due", iNum3:33, Date4:"2022-09-21", fBool5F:false, fBool6T:true, szImg7:"file://D:/Prove_JS/_Viola/Res/image0.jpg", Arr8:[1,2,3]};

In seconda battuta si potrà accedere ad un particolare campo del record specificando l'indice corrispondente.

Formati File.

La nomenclatura seguente non concerne la rappresentazione dei dati nella memoria RAM, ma quella degli stessi sotto forma di file su disco.

JSON

I file in formato JSON sono costituiti da stringhe di testo come quelle degli esempi precedenti (vedi Arr, as_, Obj, aRcd, aObj, asRcd, asObj) con due importanti differenze:
  • l'omissione della dichiarazione del nome della variabile,
  • l'omissione del punto e virgola finale.

Esempio:
Immaginando di volere salvare in un file esterno i valori della struttura dati S_asRcd bisognerà inserire semplicemente
{
"anatra_0":[41,false,"duck_3","2009-09-01","S_asRcd_5"],
"cane":[70,0,"dog","2021-01-02",""],
"cavallo":[53,true,"horse","2004-04-03",""],
"foca":[19,true,"seal","2000-10-04",""],
"gallo":[37,1,"cock","1988-08-05",""],
"gatto":[21,1,"cat","2017-02-06",""],
"lupo":[6,0,"wolf","2007-07-07",""],
"oca":[115,true,"goose","1996-06-08",""],
"rana":[84,false,"frog","2015-05-09",""],
"topo":[72,false,"mouse","2023-03-10",""]
}
omettendo il nome della variabile che non fa parte dei dati e il punto e vergola finale.

CSV.


Comma Separated Values (CSV).


File di testo costituito da liste di valori separate da virgole (o da altri caratteri convenzionali).


NOTA:
Il formato CSV è il tipico risultato che si ottiene stampando una lista di valori  in linguaggio C utilizzando una printf in un ciclo for.

Fixed Length Records (FLR).

Nei record di lunghezza fissa ciascun campo occupa sempre lo stesso numero di caratteri per cui stampando i record uno per riga i campi risultano perfettamente incolonnati.
Le funzioni printf/fprintf del C-Language (e degli altri linguaggi compatibili POSIX) sono ottimizzate per la stampa/salvataggio di dati sotto forma di record di lunghezza fissa.

Ad esempio l'istruzione

printf("%10d %6d %3d %3d %3d %3d %6d %6d\n", x1, x2, x3, x4, x5, x6, x7, x8);

potrebbe produrre un tabulato come quello sottostante.
1583403450      1   0   0   0   1      0      0
1583403451      2   0   0   0   2   2141   2141
1583403452      3   0   0   0   3   5469   5469
1583403453      4   0   0   0   4   2492   2492
1583403454      5   0   0   0   5   2230   2230
1583403455      6   0   0   0   6   1827   1827
1583403456      7   0   0   0   7   1493   1493
1583403457      8   0   0   0   8   1462   1462
Si osservi che nel caso suddetto è superfluo separare i campi usando virgole "," o altri segni di punteggiatura perché i valori corrispondenti ai diversi campi sono facilmente individuabili poiché occupano una posizione fissa nel tabulato.
Il campo "iLen" del tracciato record consente di specificare il numero di caratteri riservati per i valori di ciascun campo quando viene impiegato il formato FLR.

NOTA.

Text.

Generico file di testo non formattato.

La suddetta opzione, elencata nella lista dei possibili formati dei file che si possono caricare, fa si che il testo selezionato venga caricato e quindi elaborato conteggiando le parole ivi contenute.

La routine di caricamento restituisce quindi la tabella delle frequenze delle parole contenute nel testo.

Blob.

Il termine BLOB, dall'inglese Binary Large Object Block, designa un generico file in formato binario.

L'opzione BLOB consente di caricare il contenuto di immagini o altri tipi di file trasformandole in tabelle di dati a campi di lunghezza fissa.

OLS.

Il formato OLS prevede la possibilità di salvare nello stesso file sia i dati della collezione che il corrispondente tracciato record (layout).


NOTA:

Poiché i file OLS contengono al loro interno più collezioni di dati, per siscurezza, queste vengono sempre caricate in modalità Read Only. Nel caso in cui l'operatore desideri modificare il contenuto di una collezioni di dati dovrà selezionarla individualmente e disabilitare il flag abilitando le modifiche.
