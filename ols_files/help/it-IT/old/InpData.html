Select Collection

Toolbar di "Select Collection".


La finestra di dialogo "Select Collection" consente di selezionare una collezione di dati già in memoria o di caricarla dall'esterno.

Il modulo (form) presentato all'utente dipende dalla sorgente (Source) selezionata.

Le possibili sorgenti delle collezioni di dati sono le seguenti:

  • Collection Loaded: seleziona una Collezione già caricata nella memoria del programma.
  • New: crea una nuova collezione di dati partendo da zero.
  • File: carica una collezione precedentemente salvata in un file esterno.
  • FileDDJ: dato il path del file contenente i dati o la URL del sito su cui risiede scarica la collezione di dati indicata.
  • Local Storage: carica una collezione precedentemente salvata nel Local Storage.
  • Session Storage: carica una collezione precedentemente salvata nel Session Storage.
  • URL: carica una collezione da un sito Web compatibile.
  • Global Variable: carica i valori contenuti in una variabile di memoria globale (allo scopo di debuggare un altro programma).
  • Function: crea una nuova collezione che verrà inizializzata con dei valori generati artificialmente specificando le corrispondenti funzioni.
  • TextArea: crea una nuova collezione a partire dai dati copiati nella TextArea.

- Collection Loaded

Select collection

Cliccando sul corrispondente bottone di input si apre un menu a tendina che mostra l'elenco delle collezioni di dati già presenti in memoria consentendo di selezionarla per la visualizzazione e l'eventuale modifica.

NB: Le collezioni corrispondenti ai tracciati record (layout) delle collezioni o delle User View non vengono elencate.

AS

Imposta l'abbreviazione (solitamente costituita da una singola lettera maiuscola) per la collezione selezionata.

Show collection's

Cliccando sul corrispondente bottone di input si apre un menu a tendina che mostra l'elenco delle strutture dati associate alla collezione selezionata:


  • tuples: mostra gli elementi (Tuple) della collezione sotto forma di tabella.
  • Layout XDB.aFld: mostra il tracciato record (Layout) della collezione originale (Read Only).
  • Layout UsrView.aFld: mostra il tracciato record (Layout) della vista utente (User View) consentendo eventualmente di modificarlo.
  • XDB. mostra il descrittore XDB associato alla collezione ovvero i valori degli attributi e dei metodi del corrispondente oggetto (Read Only).
  • UsrView: mostra il descrittore UsrView associato alla collezione ovvero i valori degli attributi e dei metodi del corrispondente oggetto (Read Only).

Read Only

Il Flag ReadOnly consente, quando è permesso, di abilitare/disabilitare la modifica delle voci del modulo che verrà aperto.

Record Layout (aFld)

Cliccando sul bottone di input si può selezionare il tracciato record (layout) della nuova collezione fra quelli disponibili, in alternativa si può ricorrere all'opzione AutoDetect lasciando al programma il compito di ricostruire il tracciato record (layout) della collezione selezionata.

Embed

Il formato JSON prevede che tutte le strutture dati siano contenute all'interno di una coppia di parentesi quadre ("[]") o graffe ("{}"), quando si editano i file è però prassi comune cancellarle. La funzione embed tenta di ripristinare automaticamente le parentesi cancellate.

Transpose

Flag abilita/disabilita la rappresentazione dei dati della collezione in modalità trasposta scambiando le righe con le colonne e viceversa.

Field name in the first line

L'opzione considerata viene impiegata in combinazione con l'opzione AutoDetect.

Memorizzando i dati di una collezione in un file di testo sotto forma di tabella (kind aRcd) si usa spesso scrivere i nomi dei campi nella prima riga da usarsi come didascalia.

Il flag considerato abilita/disabilita l'utilizzo dei valori dei campi presenti nella prima riga come nomi dei campi.


NOTA:

questa opzione è utilizzabile solo nel caso di collezioni di tipo aRcd o asRcd.


NOTA:

Nel caso di asRcd il record contenente i nomi dei campi deve corrispondere alla chiave: _szKey_


Esempio:

{
"_szKey_":["szNm1", "iNum2", "fBool3"],
"key0":["asRcd", 100, true],
"key1":["bbb", 101, true],
"key2":["ccc", 102, true], 
"key3":["ddd", 103, true]
}

Manage "undefined fields"

Il linguaggio JavaScript consente di inizializzare array e record compilando solo le voci essenziali per cui è possibile che molti campi risultino indefiniti.

Il Flag considerato abilita/disabilita la gestione automatica dei campi "undefined".

- Function

Function

La voce Function consente di creare ed inizializzare una collezione di dati inserendo i valori di una funzione.

Ad esempio impostate:

  • min = 0
  • Max = 100
  • Step = 1
  • Collection Name = prova
  • Kind = a Rcd
  • Record Layout (aFld) = G_aFld_Func
  • e Confermate.


Il programma creerà automaticamente una collezione di 100 record i cui valori sono indefiniti (undefined).


Per inizializzare gli elementi della collezione basterà impostare le voci szCode del tracciato record.

  • Cliccate sulla voce Select Collection della barra dei menu.
  • Selezionate la voce Collection Loaded nella lista Source.
  • Selezionate la collezione corrispondente alla funzione nella lista Select Collection.
  • Selezionate la voce Layout UsrView.aFld nella lista Show Collection.
  • Confermate.
Apparirà una tabella consistente di due righe.
Editate le due rughe spuntando le caselle del campo fRecalc.

Cliccate nuovamente su Select Collectione confermate.
Il programma inizializzerà automaticamente i 100 record della collezione inserendo i valori della funzione y = x2 + 3x + 5.
Volendo tabulare un'altra funzione basta modificare la formula nel campo szCode del tracciato record.

NOTA: si osservi che la variabile j cui si fa riferimento nella formula corrisponde all'indice posizionale dell'elemento (riga) all'interno della collezione pertanto il valore della variabile indipendente x, che compare nella prima colonna, dovrà essere espresso (calcolato) in funzione di j.

Per visualizzare la funzione.

Cliccare sulla voce Plot della barra dei menu.
impostare i valori:
width = 100
height = 480
-1 1 100 10000 verde nero 2.
