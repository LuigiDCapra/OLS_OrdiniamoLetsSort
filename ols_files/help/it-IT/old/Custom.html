Sviluppo di routine personalizzate

L'impiego di JavaScript per manipolare le collezioni di dati, lungi da essere qualcosa di accessorio, è stato invece il punto di partenza per lo sviluppo di questo programma.
Ordiniamo/Let's Sort (OLS), precedentemente chiamato DDJ, è nato infatti come libreria per la manipolazione di dati strutturati in JavaScript. In un secondo tempo si è poi dotato di una interfaccia grafica allo scopo di facilitarne l'uso da parte degli utenti interessati ad adoperare le suddette librerie nell'analisi e nella manipolazione di collezioni di dati memorizzati nel formato interno di JavaScript oppure salvati in file sotto form ai stringhe JSON.
Il modulo test.js di Ordiniamo/Let's Sort (OLS) è stato creato allo scopo di facilitare l'integrazione di routine personalizzate; i tasti Test1 e Test2 nella barra dei menu consentono di richiamare le corrispondenti routine sviluppate dall'utente.

Tasti di Test.

 I tasti Test1 e Test2 nella barra dei menu della vista Tabella sono agganciati rispettivamente alle funzioni $Test.U_Test1 e $Test.U_Test2 definite nel file test.js.

Funzioni di Test.

L'utente programmatore è libero di modificare il codice delle funzioni $Test.U_Test1 e $Test.U_Test2 definite nel file test.js integrando il suo codice allo scopo di provarlo.

Esempio: una generica routine personalizzata.


/*-----U_Test1 --------------------------------------------------------
*
*/ 
function U_Test1()
{
  var UsrView0 = CL_UsrView0.F_UsrView_Selected();
  var XDB0  = UsrView0.XDB0;
  var Coll0 = XDB0.Coll0;
  var Rcd0;
  var iTmp;
  
  alert("Inizio Test");
  for (let i = 0; i < Coll0.length; i++) {
      Rcd0 = Coll0[i];
      iTmp = Rcd0[1];
      if (iTmp < 0) {
         iTmp = -iTmp;
         Rcd0[2] = iTmp;
      } /* if */

      Coll0[i] = Rcd0;
  } /* for */

  alert("Fine Test.");
  $Table.U_Display_Table();
} /* U_Test1 */
Data una collezione di dati memorizzati sotto forma di array di record il codice precedentemente illustrato:
  1. ottiene di descrittore della UserView della vista selezionata ( var UsrView0 = CL_UsrView0.F_UsrView_Selected(););
  2. estrae il descrittore della Collezione originale ( var XDB0  = UsrView0.XDB0;);
  3. ottiene l'accesso ai valori della Collezione (var Coll0 = XDB0.Coll0;);
  4. segnala l'inizio del Test;
  5. quindi elabora in sequenza i valori della collezione Coll0 in base all'ordine in cui sono stati memorizzati nella collezione originale;
  6. All'interno del ciclo i record vengono selezionati uno alla volta e assegnati alla variabile Rcd0 (Rcd0 = Coll0[i];);
  7. la routine estrae poi il valore del campo 1 assegnandolo alla variabile iTmp;
  8. controlla la negatività del valore considerato
  9. se il valore di iTmp è negativo viene reso positivo è assegnato al campo 2.
  10. Alla fine del ciclo, la routine segnala la terminazione del processo e
  11. aggiorna i valori mostrati sullo schermo ($Table.U_Display_Table();).

Principali funzioni.

CL_UsrView0.F_UsrView_Selected();

Il metodo CL_UsrView0.F_UsrView_Selected(); restituisce il descrittore della vista utente selezionata ovvero della UserView selezionata della collezione selezionata.

UsrView0.XDB0

L'attributo UsrView0.XDB0 è il descrittore contenente i metadati della collezione originale.

XDB0.Tup_Sel

L'attributo XDB0.Tup_Sel punta all'ultima Tupla selezionata (ovvero all'ultimo record o all'ultima riga selezionata).
Tale considerazione è degna di nota poiché il fatto che XDB0.Tup_Sel punti all'ultima Tupla selezionata implica che:

ogni modifica apportata ai valori di XDB0.Tup_Sel si traduce automaticamente in una modifica dei valori del corrispondente record nella collezione.

UsrView0.jaFld1

Il valore dell'attributo UsrView0.jaFld1 individua univocamente la colonna della tabella selezionata (nella vista utente).

UsrView0.aFld1[UsrView0.jaFld1]

L'attributo UsrView0.aFld1[UsrView0.jaFld1] è costituito dai valori dei metadati che caratterizzano il campo del record selezionato, tenendo conto che nell'ambito della vista utente i campi possono essere stati nascosti o riordinati.
