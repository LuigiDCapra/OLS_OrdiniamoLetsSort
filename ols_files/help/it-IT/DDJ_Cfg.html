DDJ_Cfg
Menu di Configurazione

Voci del modulo "Config".

szUser

Nome utente che verrà inviato al server al momento del login.

szURL_Relay

URL corrispondente al server dal quale è possibile ricaricare l'applicativo in modo da soddisfare le condizioni SOP.

szjDate

Formato delle date.

Possibili valori:

  • YYYYMMDD: formato standard usato per rappresentare le date nei database.
  • DDMMYYYY: formato delle date usate nell'Europa continentale.
  • MMDDYYYY: formato date usate nei paesi anglosassoni.

szURL_Srv

URL corrispondente al server usato per caricare le Collezioni di dati.

szSrv_Knd

Tipo di server.

szDir_Data

In riferimento a szURL_Srv directory in cui risiedono le Collezioni di dati per default.

iWdt_Tab

Dimensioni della finestra virtuale. Il parametro iWdt_Tab consente di adattare le dimensioni della finestra a quelle del tracciato record.

iSmpl_Step

Seleziona il passo per la visualizzazione di righe/tuple.

Impostando "iSmpl_Step" = 1 verranno mostrate tutte le tuple (righe) presenti nella raccolta (filtrata).
Impostando "iSmpl_Step" = 2 verrà mostrata una riga sì e una no.
" " " "
Impostando "iSmpl_Step" = 10 verrà mostrata una riga ogni dieci.
Ecc.

iNnRow_Page

Numero di righe per pagina.

szLanguage: Language

Lingua selezionata.

it-IT italiano;

en-US: inglese USA.

iWdt_Icon

Larghezza delle icone in pixel.

Modificando il valore del parametro iWdt_Icon è possibile cambiare la dimensione delle icone.

La modifica diventerà operativa dopo il ricaricamento del programma.

iNn_Col

Numero di colonne usate per mostrare le immagini nella "vista icone".

iDebug

Soglia di segnalazione delle eccezioni.

Ad ogni eccezione riscontrata (errore, warning, evento di debug) è associato un livello di priorità che per default è 5.

Abbassando la soglia di segnalazione si riduce il numero degli eventi segnalati e quindi il disturbo per l'utente.

Alzando la soglia si aumentano le capacità diagnostiche del sistema, ma anche il fastidio per l'utente.


0 = Maschera tutte le segnalazioni.

5 = valore di default.

10 = abilitazione completa delle funzioni di debug.


Gli eventi con priorità 0, come gli errori, non sono mascherabili per cui vengono sempre segnalati.

fHalt_Debug

Flag che abilita/disabilita le funzioni di debug condizionali.

fSend_DebugInfo

Flag che abilita/disabilita l'invio automatico delle informazioni diagnostiche al server.

fChk_OnLine

Flag che abilita/disabilita il controllo della connessione.

fDtStru

Flag che abilita/disabilita la dichiarazione delle strutture dati ausiliarie per fini diagnostici.

fLoadExamples

Flag che abilita/disabilita il caricamento degli esempi.

fLoadRemarks

Flag che abilita/disabilita il caricamento delle annotazioni.

fInitialize

Flag che abilita/disabilita il caricamento del file di inizializzazione.

fEnaPeriodic

Flag che abilita/disabilita l'esecuzione delle funzioni periodiche.

fIcons

Flag che abilita/disabilita l'utilizzo delle icone invece dei bottoni con il nome delle funzioni.

fVoice_Error

Flag che abilita/disabilita la lettura ad alta voce dei messaggi di errore.

Comandi della finestra di dialogo.

X.

Cliccando sul simbolo "X" che compare in alto a destra si chiude la finestra di dialogo, annullando tutte le modifiche.

Confirm.

Cliccando sul pulsante "Confirm" nella fascia verde sottostante si chiude la finestra di dialogo tornando alla rappresentazione della Collezione selezionata.

Comandi della barra delle icone.

reset
Reset

Ripristina i valori iniziali dei flag.

aiuto
Help file
