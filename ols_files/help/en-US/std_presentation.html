Ordiniamo/Let's Sort (OLS) - Presentation


Ordiniamo/Let's Sort (OLS) is an open-source application, written in JavaScript, for exploring and manipulating data collections in JSON format.
The sources of Ordiniamo/Let's Sort (OLS) are distributed under the GPL v. 3.0 license .
The files that make up the DDJ library, on which the OLS program is based, are distributed under the LGPL v. 3.0 license .


The OLS program is not a finished and functional program ready to be used by end users, but an open construction site in continuous evolution.

The Ordiniamo/Let's Sort (OLS) program is a technical tool for exploring and manipulating data collections in JSON or alternative formats.
OLS supports a wide range of sources, allowing you to load data collections from traditional files saved on disk, download tables hosted on a web server or use copy and paste.
It can also be integrated with other JavaScript programs for debugging purposes, allowing you to view the values contained in complex data structures that might be difficult to study with standard debuggers.
The Ordiniamo/Let's Sort (OLS)   program was created as part of an educational initiative aimed at allowing some students, excluded from the Italian PCTO program, to gain simulated work experience by performing work activities similar to those they might perform in a company.
The Ordiniamo/Let's Sort (OLS)  program is essentially a technology demonstrator that highlights the creative possibilities of the HTML5 + CSS3 + JS trio, aiming to capture students' attention and inspire them to develop a passion for JavaScript programming.
Given these premises, the educational objectives prevail over any other requirement, including those concerning safety,

Its use is therefore not recommended in all those contexts in which the use of OLS could constitute a breach in the security of users' computer systems..

In this regard, it should be noted that the Ordiniamo/Let's Sort (OLS)  program is the prototype of a developer tool and is intended for use by competent and responsible users who know what they are doing and take responsibility for it. It is not a tool for end users.

Ordiniamo/Let's Sort (OLS)  program is not a transactional database and does not claim to be one, but rather a datawarehouse.

Ordiniamo/Let's Sort (OLS)  software can be used in various ways:
  • to explore and/or manipulate data structures already present in the browser's memory or loaded from an external file,
  • to monitor the evolution over time of the results of the processing of a program under test,
  • to view the properties of a collection, verifying its compliance with the specifications and then possibly "cleaning" it by carrying out the so-called data cleaning ,
  • to change the data representation of a collection by converting its record layout and field format,
  • or more simply you can integrate the DDJ library into another program to use its data exploration and visualization functions.

It goes without saying that operations involving the modification and reorganization of data in a collection can be detrimental to the integrity of the collection.
Safeguarding the integrity of the data contained in the collections is the responsibility of the operator.

However, as a precaution, data collections are opened by default in Read-Only mode, so that they can be explored and viewed, but not modified. If the operator decides to switch the operating mode to enable modifications ( Read/Write mode ), he or she assumes responsibility for any damage that may be caused to the data structures.

Characteristics:
  • Support for different data representation schemes in JSON format (arrays, objects, record arrays, object arrays, record-associative arrays, and object-associative arrays).
  • Support arbitrary record tracks.
  • Ability to explore data collections by representing them in the form of tables or cards.
  • Ability to transpose rows with columns.
  • Ability to automatically convert data collections from one JSON schema to another and save them.
  • Ability to extract and rearrange subsets of data and save them.
  • Support for multimedia data collections (images, audio, video, mathematical formulas, GPS coordinates).
  • Hypertext data support . Links to records in other OLS tables or to documents accessible via the Web can be inserted into record fields.
  • Support for HTML-5 date types (date, datetime, datetime-local).
  • JavaScript type support "bigdata".
  • Multiple search filters corresponding to as many data selection methods (data query).
  • Ability to navigate within data collections by exploring them interactively.
  • Support for tables with sparse data, i.e. mostly empty columns.
  • Data driven . Ability to associate a user view with a collection's record layout , customizing its representation and user interface.
  • Support for active data collections. Manage data collections that change at runtime by receiving updates from different sources via HTTP.
  • Automatic recognition of data representation scheme .
  • Automatic data validation and coloring functions with reporting of inconsistencies.
  • Contextual help.
  • Open-source code released under the GNU GPL v. 3.0 and GNU LGPL v. 3.0 licenses.
